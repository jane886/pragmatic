1，redis 了解吗？zset 这个数据类型了解吗？

    顾名思义 REmote DIctionary Server，远程字典服务，字典即 Key - Value。
    Value 可以是：
    string（字符串），
    hash（哈希），
    list（列表），
    set（集合），
    zset（sorted set 有序集合）。
    应用：
        排行榜
    原理：
        使用的 zset 结构，zset 结构由一个字典和一个跳跃表组成。
        skiplist 是一种有序的数据结构，通过在每个节点中维持多个指向其他节点的指针，
        来达到快速访问节点的目的。
        平均 O(logN)、最坏 O(N)。还能通过顺序性操作来批处理节点。
        为什么？
            性能
            同时使用跳跃表和字典比单独使用字典或跳跃表的其中一种性能要高。
            单独使用字典，虽然查找是 O(1)，但是字典是无序的，在执行范围
            查找时先要进行排序，完成排序要花费至少O(logN)时间复杂度，
            以及 O(N) 的空间。
            单独使用跳跃表，虽然跳跃表的优点保留，但是查找的复杂度从 O(1) 上升到 O(logN)


    Redis 是一款开源的内存数据库，支持多种数据类型，包括字符串、哈希、列表、集合和有序集合等。
    有续集合是 Redis 中的一种数据类型。他是在集合的基础上增加了一个权重值，可以通过权重值来进行排序和范围查找

    在有序集合中，每个元素都有一个权重值，同时还有一个唯一的成员值。权重值可以是任意浮点数，成员值可以是任意字符串。
    有序集合中的元素是按照权重值排序的，而不是按照成员值排序的。在有序集合中，每个成员值都是唯一的，但是不同的成业值可以有相同的权重值

    有序集合支持多种操作，包括添加元素、删除元素、修改元素的权重性、查找元素、计算有序集合的基本信息等。
    在有序集合中，还可以进行范围查找操作，例如查找权重值在某个范围内的所有元素，或者查找排名在某个范围内的所有元素
    有序集合可以被用来实现排行榜、计数器等功能

    在实际开发中，有序集合是 Redis 中比较常用的数据类型之一。它可以方便地进行排序和范围查找，适用于很多场景。
    Zset 的底层是使用跳表实现的，它具有高效的插入、删除、查找等操作，而且空间复杂度比红黑树低，因此在大部分情况下，Zset 是比较优秀的数据结构选择


2，Redis 应用

    Redis是一种高性能的键值存储系统，具有丰富的功能和广泛的应用场景。下面是一些常见的 Redis 应用：

    缓存：Redis 可以作为缓存层来加速应用程序的数据访问速度。它可以将常用的数据存储在内存中，并提供快速的读写操作，减少对后端存储的访问压力。

    会话存储：Redis 可以用于存储用户会话数据。将会话数据存储在 Redis 中可以提高性能和可伸缩性，同时支持分布式环境下的会话共享。

    消息队列：Redis 的发布/订阅功能可以用作简单的消息队列系统。应用程序可以将消息发布到 Redis 的频道，并订阅者可以订阅相应的频道来接收消息。

    计数器和统计：Redis 提供了原子性操作，并支持增减操作，因此可以用作计数器和统计数据的存储。它可以用于实时统计在线用户数、访问量、点赞数等。
